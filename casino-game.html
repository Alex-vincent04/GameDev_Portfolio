Round Timing
  • 0 – 10 s: Wheel spinning, no bets allowed
  • At 10 s: "No more bets" — wheel animation starts
  • 10 – 20 s: Betting phase — bets accepted
  • At 0 s: Winner determined, payouts applied
Win Multiplier System

When the player wins, a secondary random bonus (0–11) is drawn to determine the payout multiplier. Bonus index 11 wraps back to 1×.

Payout Formula
// Winner() — simplified winner = Random.Range(0, 12); // winning slot WinBonus = Random.Range(0, 12); // bonus index if (slots[winner] > 0) // player had a bet here { multiplier = WinBonus + 1; // 1x – 11x (index 11 wraps to 1x) WinBet = slots[winner] * multiplier; Balance += WinBet; }

Example: Bet of 500 on slot 7 wins with Bonus 4 → Multiplier 5× → Payout 2,500.

Spinning Wheel Coroutines

Three coroutines (SpinWheel1, SpinWheel2, SpinWheel3) run in parallel. Each cycles through slot images every 0.1 s while Timer > 0, creating the illusion of spinning reels. Wheel 3 displays the bonus multiplier slot rather than the main winner.

// SpinWheel1 — simplified IEnumerator SpinWheel1() { int i = 0; while (Timer > 0) { wheelImage1.sprite = slotSprites[i % 12]; i++; yield return new WaitForSeconds(0.1f); } // Timer == 0 — settle on winner wheelImage1.sprite = slotSprites[winner]; }

Game Flow

Round Lifecycle
01
Betting Phase — 20 → 10 seconds
All bet buttons enabled. Player selects chip denomination then places bets on individual slots or uses quick-bet shortcuts (Row, Column, Red, Black, Odd, Even, Random). Balance is deducted immediately on each placement.
02
No More Bets — At 10 seconds
"No More Bets" audio plays. buttonsEnabled set to false. Wheel() called — launches three spinning coroutines. Bets are locked into activeBetSlots.
03
Wheel Spinning — 10 → 0 seconds
Three reel images cycle through all 12 slot sprites every 0.1 s, giving the visual effect of a spinning wheel. Spin audio plays continuously.
04
Winner Determination — At 0 seconds
Winner() fires: draws a random winner (0–11) and a random WinBonus (0–11). Wheels settle on the winning images. Payout is calculated and added to Balance if player had a bet on the winner.
05
Result — 5 second display
If player won: Win page shown, win audio & announcement audio play, winnings displayed. If lost: Loss result shown silently. History bar updates with new winning slot.
06
Reset
All slot arrays cleared. TotalBet reset to 0. buttonsEnabled restored. Timer reset to 20. Round begins again from step 01.

History System

After every round, UpdateHistory() shifts all history slot images one position and inserts the new winning slot image at position 0. This maintains a visible log of the last 12 rounds' results on-screen, giving players pattern-spotting context.

UI Elements
  • Timer Display — MM:SS countdown
  • Balance Display — live player funds
  • Total Bet Display — aggregated wager this round
  • Slot Bet Labels — per-slot chip amount (12 labels)
  • Win Amount Display — payout after round
  • History Row — last 12 winning slots as images
  • Win Page — full-screen result overlay shown for 5 s
  • Wheel Reels — 3 animated reel images (Wheel 3 shows bonus multiplier)
