2D Runner

An endless 3D-perspective runner where the player dodges obstacles and collects coins on a looping road. Features a power-up system, double-jump mechanic, and fully modular C# architecture.

Gameplay Mechanics

Player Movement
  • Move left / right with A · ← and D · →
  • X-axis clamped to ±4.75 units — keeps player on road
  • Movement speed configurable via speed parameter
  • 2× speed available via Speed2x power-up
Double-Jump System
  • Key: Space
  • Up to 2 consecutive jumps before landing
  • Jump force: 5 units impulse (Rigidbody AddForce)
  • Max height cap at 3f — prevents runaway ascent
  • Jump count resets on planeCollision tag contact

Power-Up System

All four power-ups share a single 10-second countdown timer. Collecting a new power-up resets the timer. When it reaches zero every flag is cleared simultaneously.

Power-UpTagEffectDuration
ShieldPowerupDestroy obstacles on collision instead of dying10 s
2× PointsPoint2xEach coin collected is worth +2 instead of +110 s
2× CoinsCoin2xSpawner instantiates double the normal coin count10 s
2× SpeedSpeed2xAll moving objects (obstacles, coins, road) travel at 2× speed10 s
Scoring
  • Base coin — +1 point
  • Coin with Point2x active — +2 points
  • Score is displayed in real-time on a UI Text element
  • Score persists until the game-over scene loads

Spawn Positions

Obstacles

  • X: Random between −3.35 and 3.35
  • Y: 0.5 (ground level)
  • Z: −5.209 f (just ahead of player)

Coins & Power-Ups

  • X: Random between −3.35 and 3.35
  • Coins Y: 2.2 f (mid-air)
  • Power-Ups Y: 1.0 f
  • Z: Random −15 to 5 f
Road Recycling RoadMovement.cs

Road segments move backward continuously along Vector3.back. When a segment's Z position drops below −10 f it is teleported to Z: 24.1 f. This creates a seamless infinite-road illusion with zero additional instantiation cost.

Object Cleanup Destroy.cs · CoinMovement.cs

Every spawned object checks its Z position each frame. Objects past Z < −30 f are destroyed immediately, keeping the scene's active object count stable throughout a session.

Speed2x Code Path
// Destroy.cs and ObstacleRotation.cs if (player.Speed2x) transform.Translate(Vector3.back * Time.deltaTime * Speed * 2); else transform.Translate(Vector3.back * Time.deltaTime * Speed);

// Player.cs — Update() StartTimer -= Time.deltaTime; if (StartTimer <= 0) { Point2x = false; PowerUp = false; Coin2x = false; Speed2x = false; }

Game Flow

Scene Layout

Startup Sequence
1. Scene 1 (SampleScene) loads 2. Player spawns at world origin 3. CreateCopy.Start() registers all InvokeRepeating spawn timers 4. RoadMovement begins translating road segments backward 5. First obstacle appears after 2-second delay 6. Score UI initialises at 0
Per-Frame Loop
01
Input & Movement
Read A / D / Arrow keys → translate player horizontally. Clamp X to ±4.75. Read Space → jump if JumpCount < 2.
02
World Update
Road segments, obstacles, coins, and power-ups all translate Vector3.back each frame. Speed doubles if Speed2x is active.
03
Object Cleanup
Each moving object checks its Z position. If Z < −30 f → Destroy(gameObject).
04
Power-Up Timer
StartTimer decrements by Time.deltaTime. When ≤ 0 all power-up flags are set to false.
05
Score Display
UI Text is updated every frame with the current Score value.
