Card Flip

A Unity memory-matching game where players flip cards to find 8 matching pairs within 60 seconds. Features a 3-second preview phase, coroutine-driven flip animations, Fisher-Yates shuffling, and modular scene management across main menu, gameplay, and result screens.

Board Layout

16 cards are arranged in a 4×4 grid. Each card has a hidden ID (0–7); every ID appears exactly twice. The board is shuffled at game start using the Fisher-Yates algorithm — pairs shown below as matched examples.

Matching Rules
  • Only 2 cards may be face-up (unmatched) at any time
  • Input is locked while the match-check coroutine runs
  • IDs compared with simple integer equality — no string or name matching
  • Matched cards show a dimmed colour and become permanently unclickable
  • Unmatched pair both flip back after 0.7 s delay
Win & Loss Conditions

Win

  • matchedPairs reaches 8
  • All 16 cards are face-up and dimmed
  • Scene 2 (Win screen) loads immediately

Timeout

  • Timer reaches 0 before 8 pairs found
  • Scene 2 (Result screen) loads regardless
  • No score/penalty system — pure time challenge

Grid3D.cs — 3D Layout Engine

Auto-positions all child card objects in a centred grid. Reads card dimensions from the first card's renderer so spacing adapts to any card size.

// Arrange() — simplified rows = Ceil(cardCount / columns) xSpace = cardWidth × spacingMultiplier zSpace = cardHeight × spacingMultiplier xOffset = (columns - 1) × xSpace / 2 // centres grid on X zOffset = (rows - 1) × zSpace / 2 // centres grid on Z for each card i: col = i % columns row = i / columns position.x = col × xSpace - xOffset position.z = -row × zSpace + zOffset

IEnumerator Flip(bool showFront) { animating = true; // Phase 1: compress card (150 ms) // Scale X: 1.0 → 0.05 float t = 0; while (t < 0.15f) { transform.localScale = new Vector3( Mathf.Lerp(1f, 0.05f, t / 0.15f), 1f, 1f); t += Time.deltaTime; yield return null; } // Midpoint: swap visible face front.SetActive(showFront); isFlipped = showFront; // Phase 2: expand back (150 ms) t = 0; while (t < 0.15f) { transform.localScale = new Vector3( Mathf.Lerp(0.05f, 1f, t / 0.15f), 1f, 1f); t += Time.deltaTime; yield return null; } transform.localScale = Vector3.one; animating = false; }
Input Validation Chain

Multiple guards exist to prevent invalid flips. Each check is a separate responsibility — clean separation of concerns.

01
Mouse Click Detected
CardInput.Update() fires a raycast every frame on left-click. If no card collider is hit, nothing happens.
02
Card.OnClick() Guards
Returns early if: isMatched is true / animating is true / card is already flipped (same card clicked twice).
03
GameManager.SelectCard() Guard
Returns early if inputLocked is true (match-check coroutine is running) or previewDone is false (still in preview phase).
04
Second Card Accepted
inputLocked set to true. CheckMatch() coroutine launched. No further clicks accepted until coroutine completes.
Timer System
  • Timer hidden during the 3-second preview phase
  • Counts down every frame: timer -= Time.deltaTime
  • Displayed as TIME : XX using Mathf.CeilToInt() — always rounds up
  • When timer ≤ 0 → LoadWinScene() called immediately regardless of match progress

Full Round Lifecycle
01
Initialisation
GameManager.Start() — builds ID list [0,0,1,1,…,7,7], Fisher-Yates shuffles it, instantiates 16 card prefabs at origin, calls Grid3D.Arrange() to position them, calls Init() on each card.
02
Preview Phase — 3 seconds
PreviewRoutine() coroutine calls FlipFront() on every card. Timer hidden. After 3 seconds all cards call FlipBack(). Timer shown and begins countdown.
03
Player Selects First Card
Click → raycast → Card.OnClick() passes guard checks → GameManager.SelectCard() stores card in firstCard and calls FlipFront().
04
Player Selects Second Card
Same input path. Stored in secondCard. inputLocked = true. CheckMatch() coroutine starts — no further input accepted.
05
Match Check — 0.7 s delay
Both cards visible. After 0.7 s: if firstCard.cardID == secondCard.cardID → SetMatched() on both, matchedPairs++. Otherwise → FlipBack() on both.
06
Win Check
If matchedPairs == 8 → all pairs found → LoadWinScene(). Otherwise unlock input and return to step 03.
07
Timeout
GameManager.Update() detects timer ≤ 0 → LoadWinScene() regardless of match count. Scene 2 handles both win and timeout presentation.
Data Flow Summary
CardInput.Update() → Physics.Raycast → Card.OnClick() [guard: not matched, not animating, not flipped] → GameManager.SelectCard() [guard: not inputLocked, previewDone] → first card? store firstCard, FlipFront(), return → second card? store secondCard, FlipFront() inputLocked = true StartCoroutine(CheckMatch()) yield 0.7 s compare IDs MATCH → SetMatched() × 2 → matchedPairs++ matchedPairs==8? → LoadWinScene() NO MATCH → FlipBack() × 2 inputLocked = false GameManager.Update() → timer -= deltaTime → timer ≤ 0 → LoadWinScene()
Architecture Highlights
  • Separation of concerns — input (CardInput), state (Card), logic (GameManager), layout (Grid3D) each in their own class
  • No Animator — flip achieved with a lightweight coroutine X-scale squeeze; zero overhead from animation state machines
  • Integer ID matching — O(1) comparison, no string parsing or name-based matching
  • Multiple input guards — layered validation prevents all edge cases (double-click, click-during-animation, click-during-check)
  • Self-centering grid — Grid3D computes offsets from card dimensions; changing columns or prefab size automatically re-centres
  • Dynamic difficulty — adjusting gameTime or spacingMultiplier in Inspector requires no code changes
